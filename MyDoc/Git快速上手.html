Git快速上手

CentOS7 安装Git

通常我们习惯在Linux上创建远程仓库。现在我们先安装Git，并配置一个Git专用的用户名。

yum install git -y  # yum安装Git

useradd git2020  # 创建一个用户
passwd git2020  # 为这个用户配置密码

Windows 安装Git

可以从Git的官方网站下载，https://git-scm.com/，也可以使用百度搜索到的Git软件。

安装Git软件时，直接使用官方默认的选项即可。安装完成后在随便的一个文件夹空白处右键，你都可以看到Git Bash Here与Git GUI Here，这两个我们之后再说。

创建远程仓库

回到Linux上，我们先创建一个远程仓库，注意我们要使用的是git2020这个用户，为了防止一些权限问题，最好先su git2020，或者在创建完成后chown git2020:git2020 -R 仓库目录，来修改所有者。

su git2020  # 这里我们直接跳到这个用户上

git init --bare /home/git2020/example_1.git  # 我们在/home/git2020/创建一个叫example_1.git的仓库

chown git2020:git2020 -R /home/git2020/example_1.git  # 如果你没有跳到这个用户上就要执行这个命令来修改所有者

配置Git基本信息

回到Windows上，在随意文件夹空白处右键，选择Git Bash Here，开启Git命令行。

# 右键文件夹空白处选择Git Bash Here
# 之后要在开启的命令行里执行

git config --global user.name "zsc"  # 配置Git全局用户名
git config --global user.email "zsc@email.com"  # 配置Git全局邮箱信息

克隆远程仓库

回到Windows上，我们选一个合适的文件夹用来存放一会要克隆的远程仓库。
打开文件夹，然后在文件夹空白处右键Git Bash Here，启动Git的命令行，在哪右键，这个命令行的工作目录也会跟着跳转。

# 右键文件夹空白处选择Git Bash Here
# 之后要在开启的命令行里执行

git clone git2020@192.168.200.129:/home/git2020/example_1.git  # @前面是用户名，后面是服务器ip，:后面是远程仓库的绝对路径
git2020@192.168.200.129's password:  # 提示我们要输入git2020的密码

# 这时这个远程的仓库就已经下载到本地了，我们发现本地的仓库文件夹名为example_1，且里面有一个隐藏文件夹.git

添加文件并提交到本地仓库

依然在Windows上，进入到下载好的本地仓库文件夹example_1，随意的创建一个文件，例如readme.txt，并写入一些内容。
然后在空白处右键Git Bash Here

# 右键文件夹空白处选择Git Bash Here
# 之后要在开启的命令行里执行

git add .  # 添加当前目录的所有修改到暂存区，这个命令也可以只添加输入的文件
git status -s  # 查看当前目录文件状态
# 显示“A  readme.txt”，表示readme.txt是添加的状态

git commit -m "add file readme.txt"  # 提交修改到本地仓库，双引号内是日志信息

git log --oneline --graph # 显示提交日志，--oneline是简洁模式，--graph是显示拓扑图

将本地仓库内容推送到远程仓库

依然在Windows上。
提交只针对本地仓库，若想远程仓库也拥有同样的修改，需要推送到远程仓库。
进入到本地仓库文件夹example_1，然后在空白处右键Git Bash Here

# 右键文件夹空白处选择Git Bash Here
# 之后要在开启的命令行里执行

git push origin master  # 将修改推送到远程仓库，origin远程主机名（默认），master为分支名（默认），注意需要输入密码

创建一个分支并推送

依然在Windows上。
进入到本地仓库文件夹example_1，然后在空白处右键Git Bash Here

# 右键文件夹空白处选择Git Bash Here
# 之后要在开启的命令行里执行

git branch test  # 基于当前的分支，创建一个名为test的分支
git branch  # 查看当前所有分支

git checkout test  # 切换到test分支上

# 这时我们修改readme.txt文件，随意的添加一些内容

git add .  # 添加当前目录的所有修改到暂存区
git status -s  # 查看当前目录文件状态
git commit -m "modify readme.txt for branch test"  # 提交修改到本地仓库，双引号内是日志信息

git push origin test  # 将test分支也推送到远端

合并分支，解决冲突

依然在Windows上。
进入到本地仓库文件夹example_1，然后在空白处右键Git Bash Here

# 右键文件夹空白处选择Git Bash Here
# 之后要在开启的命令行里执行

git checkout master  # 切换到master分支上
# 这时我们发现由于我们切换了分支，test分支修改的readme.txt的内容不见了

# 再次修改readme.txt文件，随意的添加一些内容，注意和test分支添加的要不一致才有冲突要解决

git add .  # 添加当前目录的所有修改到暂存区
git status -s  # 查看当前目录文件状态
git commit -m "modify readme.txt for branch master"  # 提交修改到本地仓库，双引号内是日志信息

git merge test  # 将test分支合并到当前分支（master）上
# 这时Git命令行给我们提示，Automatic merge failed（自动合并失败），此时我们需要手动打开readme.txt文件，清理合并差异。

<<<<<<< HEAD
add for master
=======
add for test
>>>>>>> test

# 我们看到以上的内容，add for master和add for test是我分别为master和test分支提交的内容。

add for merge

# 将冲突的部分修改成想要的内容，注意要删掉<<<<<<< HEAD和>>>>>>> test等多余的部分。

git add .  # 添加当前目录的所有修改到暂存区
git status -s  # 查看当前目录文件状态
git commit -m "merge test to  master"  # 提交修改到本地仓库，双引号内是日志信息

git log --oneline --graph # 显示提交日志，--oneline是简洁模式，--graph是显示拓扑图

# 这样我们就将分支合并起来了，当然，也可以先解决冲突，再合并分支。合并结束后，被合并分支不会发生改变。

git branch -d test  # 删除test分支

删除文件，同时取消对文件的跟踪

依然在Windows上，我们在文件夹内删除文件不会取消对文件的跟踪，所以需要通过命令操作这个文件。
进入到本地仓库文件夹example_1，然后在空白处右键Git Bash Here

# 右键文件夹空白处选择Git Bash Here
# 之后要在开启的命令行里执行

git rm --cache readme.txt  # 删除对文件的跟踪，但不真正的删除真实文件
git rm readme.txt  # 删除对文件的跟踪，同时删除真实文件

获取远程仓库内容到本地

依然在Windows上，我们将获取远程仓库的最新更新
进入到本地仓库文件夹example_1，然后在空白处右键Git Bash Here

# 右键文件夹空白处选择Git Bash Here
# 之后要在开启的命令行里执行

git fetch origin master  # 获取远程仓库master分支的最新更新，但不影响本地仓库
git merge origin/master  # 将远程master分支合并到本地当前分支上，若出现冲突需要手动解决并提交

总结

git push需要连接到服务器，需要输入密码。
git commit只针对本地，不需要网络。

Git GUI Here，Git Bash Here的界面版本，更加的简洁清晰，这里不做过多介绍。
